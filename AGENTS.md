## 项目定位

本项目是“知网文献综述助手”的后端 API 服务，服务于浏览器插件项目 `cnki-extension-ai-export`。

后端负责用户登录、额度校验、DeepSeek API 调用、快速综述和深度综述报告生成。浏览器插件负责知网页面采集、本地文献集合管理、PDF 文本提取、报告下载和用户交互。

## 技术栈约定

- 运行时：Node.js
- 语言：TypeScript
- Web 框架：Fastify
- 数据库：SQLite
- ORM：Prisma
- 部署目标：腾讯云服务器
- 进程管理：优先使用 PM2；后续可按需要切换 Docker

## 功能边界

后端应该做：

- 用户注册、登录和鉴权。
- 快速综述额度校验与扣减。
- 深度综述任务额度校验与扣减。
- 调用 DeepSeek API 生成报告。
- 返回报告内容给插件端下载。
- 记录必要的任务元信息、额度流水和错误日志。

后端不应该做：

- 自动下载、抓取或绕过知网 PDF / CAJ 访问限制。
- 绕过知网登录、验证码、付费或权限控制。
- 长期保存用户上传的 PDF 原文。
- 长期保存生成后的综述报告。
- 提供论文代写、伪造引用或不可追溯内容。

## 产品模式

- 游客：可在插件端采集当前页和下载采集结果，不调用后端 AI 分析。
- 注册用户：获得免费快速综述额度。
- 付费用户：购买快速综述额度或深度综述任务包。
- 快速综述：基于题录、摘要、关键词生成初步综述素材。
- 深度综述：基于题录、摘要、关键词和用户合法获取的多篇 PDF 全文生成深度综述素材。

## 数据与隐私原则

- 默认最小化保存用户数据。
- PDF 全文和报告正文在 MVP 阶段仅用于当次生成，不做长期云端历史记录。
- DeepSeek API Key、支付密钥、数据库连接串等敏感信息只能保存在后端环境变量中。
- 不要把任何服务端密钥写入浏览器插件或前端构建产物。
- 日志中避免记录完整论文全文、完整报告正文、用户密码或令牌。

## Python 虚拟环境约定

- 如需在本项目中使用 Python 辅助脚本，必须使用项目内虚拟环境 .venv。
- 不要把 Python 依赖安装到系统环境。
- .venv、env、__pycache__ 和 *.pyc 不进入版本库。
- 当前后端主体仍为 Node.js + TypeScript，Python 仅作为后续辅助脚本环境。

## 开发注意点

- 优先保持接口简单、可测试、可部署。
- 不提前实现复杂会员体系、云端文献库、团队协作或历史报告管理。
- 所有 AI 输出必须强调“辅助整理”和“综述素材”，避免包装为“自动代写论文”。
- 快速综述和深度综述都必须依赖插件端已采集的文献集合。
- 深度综述必须依赖已采集题录来匹配 PDF，不接受无上下文的孤立 PDF 批量分析作为主流程。
- 对免费额度、接口频率、单次文献数量和输入长度做后端校验，不能信任前端传入的限制值。
